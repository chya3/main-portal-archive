# Main Portal Archive

本项目用于归档和发布多个独立的 HTML 页面。仓库本身不针对任何特定域名或具体网站，仅作为静态页面资源的统一存放与版本管理场所。

## 项目简介

`main-portal-archive` 是一个通用型 HTML 页面归档仓库。你可以将需要长期保存或对外展示的独立网页文件（如信息页、导航页、工具页等）放置于此，利用 GitHub Pages 或其它静态托管服务进行发布。仓库结构清晰，便于分类管理与历史追溯。

## 目录说明

```
main-portal-archive/
├── pages/            # 存放所有归档的 HTML 页面文件
│   └── index.html    # 示例首页（可选，可自行替换或删除）
├── assets/           # 公共资源目录（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── docs/             # 项目文档或说明（可选）
└── README.md         # 本文件
```

- **pages/**：核心目录，每个 HTML 页面建议单独存放或按功能分组（如 `pages/tools/`、`pages/archive/`）。
- **assets/**：存放页面共用的样式、脚本及图片资源，避免重复引用。
- **docs/**：可放置更详细的使用说明、变更日志或部署指南。

## 页面归档说明

- 所有归档页面均为独立的 HTML 文件，相互之间无强制依赖关系。
- 页面内引用的资源建议使用相对路径（如 `../assets/css/style.css`），以确保在不同部署环境下均可正常加载。
- 如需添加新页面，请在 `pages/` 目录下新建文件或子目录，并保持命名简洁一致。
- 历史版本可通过 Git 提交记录回溯，不建议在同一文件中保留大量注释掉的旧代码。

## 维护说明

- 本仓库由维护者不定期更新，接受合理的改进建议或修复请求。
- 若需提交新页面或修改现有页面，请先创建 Issue 说明变更内容，或直接发起 Pull Request。
- 请勿在仓库中存放包含敏感信息、商业机密或违反开源社区准则的内容。
- 所有页面内容仅代表贡献者个人行为，与仓库维护者无关。

## 部署方式（可选）

你可以通过以下方式发布仓库中的页面：

1. **GitHub Pages**：在仓库设置中启用 Pages 功能，选择 `main` 分支的 `/` 根目录或 `/docs` 目录作为源。
2. **其他静态托管**：将仓库克隆到本地后，将 `pages/` 目录内容上传至任意支持静态文件的服务器或 CDN。
